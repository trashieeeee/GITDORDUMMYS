GIT első lépések 1. Telepítés: https://git-scm.com/download/win
Minden lépés tovább, a Git Basht telepítsük fel, saját munkánkat könnyíti meg.
2. C, vagy D meghajtó gyökerében dolgozzunk, egyszerűsödik a munkánk.
3. Regisztráció a GitHub.com oldalon (iskolai e-mail címmel regisztráljunk)
4. Hozzunk létre egy repository-t, egyszerű névvel.
5. Azonosítás beállítása, szükséges a Github pontos kapcsolathoz
a. git config --global user.name "Github-on megadott név"
b. git config --global user.email "Github-on regisztrált emailcím"
6. Hozzunk létre a lokális mappánkba egy repositoryt.
a. mkdir repository-name
b. cd repository-name
c. git init
(a mappa létrehozást csinálhatjuk windows alól is természetesen)
7. Távoli kapcsolat beállítása a későbbi feltöltéshez
a. # új origin nevű távoli kapcsolat beállítása
b. git remote add origin repository-link (github-ból code-ból elérhető a közvetlen link)
c. # távoli kapcsolatok lekérdezése
d. git remote -v
e. # origin nevű távoli kapcsolat frissítése
f. git fetch origin
g. # minden branch lekérdezése
h. git branch –a
8. Hozzunk létre egy fájlt a lokális munkamappába (windowsban könnyebb)
9. Adjuk hozzá a fájlt az átmenetei tárolóhoz (stage)
Egyetlen fájl hozzáadása a staging area-hoz:
a. git add file-name
Minden az aktuális mappában, vagy annak almappáiban található fájl hozzáadása a
staging area-hoz:
a. git add .
i. Piros: azon fájlok, amik nem kerültek be a staging-area-ba
ii. Zöld: azon fájlok, amik berültek a staging-area-ba
10. Státusz lekérdezése
a. git status
11. Átmeneti tárolóból a repoba való áttöltés (kommitálunk)
a. leíró szöveg megadása parancssori argumentummal
b. git commit -m "Kötelező megjegyzést tenni"
12. Eddigi verzió megtekintése
a. git log
i. Minden egyes bejegyzéshez a következőhöz hasonlót fogunk látni:
commit a661d9249509f8aa8f81f6f58129d57eb36b81d6
Author: Gergo Roth
Date: Wed Jul 6 18:36:01 2016 +0200
My first commit.
ii. A "commit" után szereplő szám egy hash érték, ami a fájlokból kerül generálásra.
iii. Az "Author:" után szerepel a változtatás végrehajtójának a neve és e-mail címe.
iv. A "Date:" után szerepel a módosítás végrehajtásának dátuma.
v. Minden alatta levő szöveg a következő bejegyzésig a commit leírása.
13. Fájl és verzió feltöltése távoli tárhelyre
a. git remote –v (kapcsolatot ellenőrizzük)
b. git push origin master
14. Távoli tárhely adatainak letöltése, helyi verzió frissítése
15. git pull origin master
